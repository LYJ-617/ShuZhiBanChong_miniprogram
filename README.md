# 数智伴宠 - 微信小程序

## 项目概述
"数智伴宠"是一款以"养宠日志"AI智能分析为核心的宠物健康管理平台小程序，结合社区、商城、问诊等服务，为用户提供一站式的宠物护理解决方案。

**核心特色**：
- 🤖 AI智能健康分析：基于养宠日志自动识别宠物健康风险
- 💬 社区互动：分享养宠经验，交流心得
- 🛒 智能商城：根据AI报告推荐合适商品
- 🏥 在线问诊：匹配专业兽医专家
- 📅 服务预约：便捷的线下服务预约

## 核心功能

### 1. 注册与宠物信息录入
- 用户注册功能
- 支持绑定多只宠物
- 标签式宠物信息录入（种类、品种、性别、生日、性格、喜好）

### 2. 首页
- **养宠日志核心功能入口**：视觉重点突出，当天未记录时显示提示语
- 日程提醒模块：疫苗、驱虫、喂药等宠物相关事项
- 服务功能入口：商城、问诊、预约、社区、我的

### 3. 养宠日志模块（核心）
- **日志发布**：
  - 选择宠物
  - 参与话题标签（#日常、#科普、#健康、#趣事）
  - 发布位置选择（宠物日志/社区）
  - 健康状态记录（粪便、食欲、精神）

- **AI健康分析**：
  - 基于关键词识别的健康风险分析
  - 风险等级评估（低/中/高）
  - 可能疾病风险提示
  - 健康建议与推荐
  - **免责声明**：确保AI建议不能替代专业医疗诊断

### 4. 社区模块
- 帖子发布与浏览
- 话题分类（日常记录、医疗知识、好物分享、科普知识）
- 互动功能：点赞、评论、收藏
- 用户主页展示

### 5. 服务模块
- **商城**：基于AI报告智能推荐商品
- **线上问诊**：匹配擅长对应领域的兽医专家
- **服务预约**：线下宠物服务预约
- 宠物选择切换，为不同宠物提供个性化推荐

### 6. AI分析报告页面
- 按宠物生成独立的健康分析报告
- 风险等级可视化展示
- 健康提示、疾病风险、推荐商品/专家/医院
- 完整的免责声明

### 7. 我的页面
- 用户信息展示
- 宠物列表管理
- 退出登录功能

## 技术栈

### 前端
- **框架**：微信小程序原生框架
- **语言**：TypeScript
- **样式**：Less
- **状态管理**：本地存储（Storage）

### 后端（待接入）
- **数据库**：MySQL 8.0+
- **API规范**：RESTful API
- **认证方式**：Bearer Token
- **AI分析**：基于关键词匹配的规则引擎（可扩展为AI模型）

## API接口设计

### 接口规范
- **Base URL**: `https://api.petcare.com/api/v1`
- **认证方式**: Bearer Token
- **数据格式**: JSON
- **状态码**: 200(成功) | 400(参数错误) | 401(认证失败) | 404(不存在) | 500(服务器错误)

### 核心接口
```
POST   /user/register          # 用户注册
POST   /user/login             # 用户登录
GET    /user/info              # 获取用户信息
POST   /user/logout            # 用户登出

GET    /pet                    # 获取宠物列表
POST   /pet                    # 添加宠物
PUT    /pet/:id                # 更新宠物
DELETE /pet/:id                # 删除宠物

GET    /log                    # 获取日志列表
POST   /log                    # 创建日志
DELETE /log/:id                # 删除日志

POST   /ai/report              # 生成AI分析报告

GET    /community/posts        # 获取帖子列表
POST   /community/posts        # 创建帖子
POST   /community/posts/:id/like      # 点赞帖子
POST   /community/posts/:id/favorite  # 收藏帖子
```

详细接口文档请查看 [API_DOC.md](./API_DOC.md)

## 开发指南

### 1. 本地开发
```bash
# 克隆项目
git clone <repository-url>

# 使用微信开发者工具打开项目
# 在微信开发者工具中导入项目目录
```

### 2. Mock数据模式
当前项目使用Mock数据模式，数据存储在本地Storage中。

切换到真实后端：
1. 修改 `utils/config.ts` 中的 `USE_MOCK` 为 `false`
2. 配置正确的 `BASE_URL`
3. 确保后端服务正常运行

### 3. 添加新页面
1. 在 `pages/` 目录下创建新页面文件夹
2. 创建 `wxml`, `ts`, `less`, `json` 文件
3. 在 `app.json` 中注册页面路由

### 4. 添加新接口
1. 在 `utils/type.ts` 中定义数据类型
2. 在 `utils/apiV2.ts` 中实现接口方法
3. 使用 `http` 实例发起请求

## 注意事项

### 开发注意事项
- 本项目当前为演示版本，数据存储在本地
- AI分析为模拟实现，实际使用需接入真实AI模型
- 所有医疗相关建议均需附带免责声明
- API接口已按照RESTful规范设计，便于后续接入真实后端

### 安全注意事项
- 用户隐私数据需要加密存储
- 敏感操作需要二次确认
- API请求需要添加Token认证
- 防止SQL注入和XSS攻击

### 法律注意事项
- AI分析结果仅供参考，不能替代专业医疗诊断
- 社区内容需要审核，防止违规信息
- 用户协议和隐私政策需要完善
- 医疗建议需要明确免责声明
